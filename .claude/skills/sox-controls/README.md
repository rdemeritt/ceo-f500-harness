# Skill: SOX Controls

## Purpose
Patterns for Sarbanes-Oxley compliance — control design, testing, documentation, and reporting for public companies.

## When to Invoke
- Designing or modifying a financial control
- Testing control operating effectiveness
- Documenting control deficiencies
- Preparing for external audit reliance on internal controls

## Key Concepts

### Control Types
- **Preventive:** Stops an error/fraud before it occurs (segregation of duties, access controls)
- **Detective:** Identifies errors/fraud after the fact (reconciliations, reviews)
- **Manual:** Human-performed (management review, approval)
- **Automated:** System-performed (system edit checks, automated matching)

### Control Documentation Required
Every key control must have:
1. Control description (what it does)
2. Control owner (who performs it)
3. Frequency (daily, weekly, monthly, quarterly)
4. Population and sample (what is being tested)
5. Evidence (what documentation proves it ran)
6. Exception process (what happens when control fails)

### SOX 404 Testing Approach
```
1. Identify significant accounts and disclosures
2. Map financial statement risks to controls
3. Determine key controls (those that prevent/detect material misstatement)
4. Test design effectiveness: Is the control designed to work?
5. Test operating effectiveness: Is it actually working?
6. Document results and exceptions
7. Report deficiencies with classification
```

### Deficiency Classification
- **Control Deficiency:** Design or operation gap; low risk of misstatement
- **Significant Deficiency:** More than inconsequential risk; report to Audit Committee
- **Material Weakness:** Reasonable possibility of material misstatement; requires public disclosure

## Compliance Notes
- Section 302: CEO + CFO certify quarterly
- Section 404: Annual management and auditor assessment
- PCAOB standards govern external auditor testing
