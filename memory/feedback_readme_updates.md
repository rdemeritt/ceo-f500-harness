name: Update README on material changes
description: Always update the README when making material changes to the harness
type: feedback
---

Update the README whenever a material change is made to the harness.

**Why:** The README is the primary reference for understanding the system; stale docs create confusion.

**How to apply:** Before committing any material change (new configuration, new agent behavior, new workflow, role changes), check if the README needs updating and include it in the same commit.
